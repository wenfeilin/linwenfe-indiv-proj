# Takeaways From 3 Experiments

- Entry Navigation:
  - When designing the UI, like for the book icon that is supposed to let you flip through the entries, **make sure to clue the user of that behavior**, like by adding hover animations, using useful symbols/arrows/icons, and/or text (maybe even a tooltip). Implementing a tutorial of how to use everything could help with this too. 
  - Definitely list the songs chosen for the entries in the calendar view.
  - Consider **alternative designs for the section that lists the month of the calendar page** (Ex: regarding the layout of "September" and "9"). This concerns readability.
  - Definitely include a different way of grouping the **entries as a list**, like a **table of contents**. Make sure to only include non-empty entries in this list. Whether the dates of these entries should be listed too here is not super important.

- Creating/Deleting Entries:
  - Don't create extra buttons if there isn't a need to. Like with the edit and done buttons that enter and exit the user into and out of edit mode, have those as one button that flip back and forth is preferred to graying out a button and creating a new button next to it to exit out of edit mode. 
  - Confirm to the user that pressing the Done button in edit mode **saves changes**. Also, have a **confirmation pop-up** for when user wants to go to the calendar mode when changes are unsaved (still in edit mode with changes made but Done wasn't pressed).
  - Use colors for buttons to make them stand out from rest of page, especially if intending to add more visual elements to the design later.
  - Customization, like having **stickers**, is definitely an important need for users. It's what makes this app more special to them/incentivizes them to use it.
  - Try to include more features if possible.

- Music Players (Monthly, One Entry):
  - Consider different designs for the music players. It might be easier to digest the UI of them if they don't have physical designs, like if they were just **boxes/rectangular widgets** with some sound wave animations, the song info, and playback controls.
  - It can be **overwhelming** to have **two music players appear in the entry mode**. Consider workarounds for that to make it less overwhelming/confusing for the user. 
  - Need a way to **exit search songs mode** if not changing the selected song (for the entry music player).
